## 開発五戒
1. **実用性（Pragmatism）の徹底：**
   機能しない壮大なインフラよりも、「今動く最小の第一原理（コード）」を優先せよ。FastAPIやRedisなどの重厚な自律バックエンドは「排除（Eliminate）」し、Google-hosted Sandbox上のPython実行（Code Execution）とManaged Agentsのステート永続化（store=True）に処理を閉じること。
2. **MongoDB MCPの積極的・自律的行使：**
   エージェントはチャットボットのように振る舞うのをやめ、ユーザーのインプットの度に必ずMongoDB MCPを叩いて、データの参照（R）、蓄積（C）、リンク結合（U）、古い情報の代謝（D）をマルチステップで実行せよ。
3. **境界線の融和（寄り添い）：**
   UIはユーザーの行動導線を邪魔しないシンプルさを追求すること。GAS（Google Apps Script）をプロキシとしたWeb-UI、もしくはSlack連携を最速で開通させること。
4. **仕様駆動開発（SDD）の遵守：**
   コードを書き散らすな。変更を行う前に、必ず `specs/` ディレクトリ内の仕様書（Markdown）と照らし合わせ、タスクに分解して1ステップずつ実装・テストせよ。
5. **ハッカソン提出要件の網羅：**
   オープンソースライセンス（Apache-2.0）の配置、リポジトリのAboutセクションの整備、3分デモに耐えうる「可視化（思考プロセスの出力）」の実装を常に念頭に置いてコーディングせよ。

function onOpen() {

function showNoiseInputDialog() {
  const html = HtmlService.createHtmlOutputFromFile('InputForm')

function sendNoiseToAgent(userInput) {
  try {
    const payload = {
      "contents": [{
        "role": "user",
        "parts": [{ "text": userInput }]
      }],
      "interaction_config": {
        "store": true // サンドボックス内の状態をセッション維持
      }
    };
    
    const options = {
      "method": "POST",
      "headers": {
        "x-goog-api-key": API_KEY,
        "Api-Revision": "2026-05-20",
        "Content-Type": "application/json"
      },
      "payload": JSON.stringify(payload),
      "muteHttpExceptions": true
    };
    
    const response = UrlFetchApp.fetch(ENDPOINT, options);
    const resText = response.getContentText();
    const resJson = JSON.parse(resText);
    
    // エージェントの回答および思考プロセスのテキストを抽出
    const agentResponse = resJson.candidates[0].content.parts[0].text;
    return agentResponse;
    
  } catch (e) {
    return "エラーが発生しました: " + e.toString();
  }
}
