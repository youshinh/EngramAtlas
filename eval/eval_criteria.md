# **Evaluation Criteria: Day 6-8 自律回復＆デバッグ (Resilience)**

## **1. 概要**

本ドキュメントは、Evaluation Driven Development (EDD) の規約に従い、Day 6-8 自律回復＆デバッグ（レジリエンスフェーズ）におけるシステムの成功基準を明確に言語化したものである。

---

## **2. 評価基準（Assertions & Grading）**

仕様書（`EngramAtlas Blueprint.md` / `AGENT.MD`）に基づき、以下の7つの評価項目を定義する。
自動評価スクリプト（`run_eval.js`）は、これらのアサーションを自動検証し、スコアを算出する。

### **項目 1: UI 疎通（Accessibility）**
- **判定条件**: `http://localhost:3000/` に GET リクエストを送信した際、HTTP ステータスコード `200` が返り、レスポンスの HTML 内に `EngramAtlas`、`noiseInput`、`toggleLanguage`、および `marked.min.js`（Markdown パースライブラリ）等が含まれていること。
- **配点**: 10点

### **項目 2: API 疎通と整合性（API Integrity）**
- **判定条件**: `/api/sendNoise` に対し、`userInput` および `lang` を含む POST リクエストを送信した際、ステータス `200` で JSON レスポンスが返り、かつ `db_id` と `response` フィールドが含まれていること。
- **配点**: 10点

### **項目 3: データ永続化（Create）**
- **判定条件**: `/api/sendNoise` の実行後、MongoDB（またはモックDB）にドキュメントが新規挿入され、一意なドキュメントID (`db_id`) が返却されること。
- **配点**: 10点

### **項目 4: エージェント思考プロセス（Gemini Integration）**
- **判定条件**: レスポンスに含まれる `response`（思考の軌跡）テキストが空でなく、かつ Markdown 形式の構造を持っていること。
- **配点**: 15点

### **項目 5: 自己組織化と双方向リンク構築（Self-Organization & Metabolism）**
- **判定条件**: 
  1. 類似する2つの思考ノイズを連続で送信した際、2つ目の Engram ドキュメントの `related_links` 配列に、1つ目のドキュメントID、類似度スコア、接続理由（`reason_of_connection`）が自動で結線されていること。
  2. 1つ目の Engram ドキュメント側も自動更新（Update）され、その `related_links` に2つ目のドキュメントIDが登録されていること（双方向リンクの確立）。
  3. `reason_of_connection` が AI によって自動生成されており、空ではないこと。
- **配点**: 20点

### **項目 6: 自律回復とトリアージ（Resilience & Self-Healing）**
- **判定条件**:
  1. `/api/sendNoise` に対し、一時的エラーシミュレーション（瞬断・429リミット）をトリガーするパラメータ `simulateError: "transient"` を付与してリクエストを送信した際、サーバーがエラーを検知し、即座にトリアージを行い、**指数バックオフ付き自動リトライ（Exponential Backoff）**を実行すること。
  2. 指数バックオフによる複数回のリトライ（待機時間が増加すること）をクリアし、最終的に自動で自己修復（Self-Healing）され、HTTP ステータス `200` で正常なレスポンスが返ってくること。
  3. レスポンスの中に、リトライが試行された軌跡（例: `[Triage: TRANSIENT]` や `Attempt` などのトリアージ・リトライ記録、または自己修復の文字）が含まれていること。
- **配点**: 20点

### **項目 7: ライセンス適合（Hackathon Guardrail）**
- **判定条件**: プロジェクトルート直下に `LICENSE` ファイルが存在し、ファイル内に `Apache License` および `Version 2.0` の記述が含まれていること。
- **配点**: 15点

---

## **3. 総合評価スコア定義**

- **100点**: 完全適合（Green / レジリエンス 成功）。自己修復機能、自己組織化、UIレンダリング、ライセンスなど、すべての要件を一貫して満たしている状態。
- **80点以上**: 主要機能動作（要改善）。自己組織化や自己修復は部分的に動作するが、リトライ履歴や指数バックオフの間隔が仕様通りでない状態。
- **80点未満**: 不適合（Fail / Red）。一時的なエラー発生時に自動リトライおよび自己修復に失敗し、システムがクラッシュまたはエラー（500/503）のまま返ってしまっている状態。
